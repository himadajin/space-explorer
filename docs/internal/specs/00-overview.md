# space-explorer 仕様書 — 概要

この文書群は、単一 HTML の artifact として実装された宇宙探索アプリ space-explorer の現在の状態を記述する。読者は本開発の経緯を知らない第三者(人間または言語モデル)を想定し、この文書群のみからアプリを正しく再実装できる詳細度で書かれている。歴史的経緯は扱わない(設計判断の根拠は 80-decisions.md にのみ現在形で記録する)。

## アプリの定義

space-explorer は、無限に広がる決定論的な宇宙を、惑星から宇宙の大規模構造(宇宙網)までシームレスにズームして探索する Web アプリである。すべての天体・構造は階層アドレス(address)によって識別され、同じ seed と同じ address からは常に同じ対象が再生成される。ユーザーは見えている対象を選んで移動でき、未訪問の address を指定して跳ぶこともでき、気に入った場所を手帳(ブックマーク)に書き留めて再訪できる。

体験の核は次の一文に要約される: 開いた瞬間に宇宙網の中におり、ズームインすると一つの惑星に降り、その address を控えれば、いつでも、誰でも、同じ惑星に再会できる。

## 実行環境

単一の HTML ファイルである。Three.js r128 を cdnjs から読み込む。ビルドツール・テストランナー・サーバー・ブラウザ永続化(localStorage 等)は使用しない。探索状態の持ち越しは address 文字列の copy と jump のみで成立する。スマートフォン縦画面が性能・レイアウトの基準である。

ファイル内部は次のセクションに区分される: core/hash、core/address、gen、(cosmic web)、(neighborhood / zoom path)、state、render、ui。core 相当(コメントマーカー CORE-BEGIN / CORE-END の間)は DOM にも Three.js にも依存しない純関数群であり、Node.js でそのまま実行して検証できる(70-verification.md)。

## 4軸の用語

以下の4軸は互いに独立した概念であり、混同してはならない。

1. ズーム体験のスケール(表示ステージ)。0 惑星 / 1 恒星系 / 2 銀河 / 3 銀河団 / 4 宇宙網。連続ズーム値 Z 上に並ぶ表示の段であり、40-zoom.md が定義する。
2. 宇宙の構造(物理的語彙)。ボイド、ウォール、フィラメント、ノード、銀河、恒星系、惑星、衛星。30-cosmic-web.md と 20-generation.md が定義する。
3. 階層アドレス(識別子)。S / W / C / G / Y / P / M の segment 列。対象の同一性は address のみが持つ。10-identity.md が定義する。
4. LOD(描画表現)。同じ対象がスケールに応じて点・スプライト・点群・詳細メッシュとして描かれる。50-art.md と 40-zoom.md が定義する。

## 規範ラベル

全文書の記述は次の三層でラベル付けされる。ラベルのない記述は文脈のラベルに従う。

- [不変] 破ると既存の address・fingerprint・手帳が無効化される契約。変更にはデータ互換性の破壊を伴う明示的な決断が必要。ビット単位の正確な再実装が要求される。
- [方式] 現在の実装選択。置換可能だが影響範囲が広い。置換する場合は関連する完了条件と検証を再設計する。
- [調整] 体験チューニングのために自由に変更できる値。変更は表示のみに影響し、identity には影響しない。

特に重要な警告: 20-generation.md に列挙されるパラメータ生成の乱数消費順・分布は、fingerprint の入力を構成するため [不変] である。「見た目の定数」に見えても、genPlanet / genProxy 内部の値は identity の一部である。一方、render 層の定数(色・サイズ・不透明度・点数)はすべて [調整] である。

## 中核不変条件(全リスト)[不変]

1. 同じ seed と同じ address からは、常に同じ対象(同じ fingerprint)が再生成される。
2. 対象の同一性は address が持つ。表示名・描画順・キャッシュ状態は同一性に関与しない。
3. cache の有無で対象の意味は変わらない(cache は純粋な高速化である)。
4. address はシリアライズと復元が可能で、往復で正規形が保存される。
5. address は全 segment を持つ必要がない。途中で終わる address はその深さの対象を指す。中間 segment の省略(スキップ階層)も同一性として有効である。
6. 周辺候補・子の列挙は正規個数(canonical count)に限定されるが、jump は正規範囲外の index も解決できる(未訪問到達性の優先)。範囲外対象の同一性・再訪問性も保証される。
7. 二つのスケールに現れる視覚的性質(色・向き・構成)は、単一の導出関数から引かれる(親子整合)。
8. 生成はローカル評価のみで行われ、周辺の必要領域だけをストリーミング生成できる。全域の事前生成は存在しない。
9. 表示品質(detail)は表示密度・解像度のみに影響し、identity には一切影響しない。同じ seed・address・品質なら同じ絵になる。
10. ブックマークは {address, fingerprint} の組であり、セーブデータではなく座標である。アプリを閉じれば消えるが、宇宙は消えない。
