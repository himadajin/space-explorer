# 40 — zoom: 5ステージ積層モデル

ズームは画面切り替えではなく、座標系の連続合成である [方式]。カメラは奥行き方向に一切動かず、各表示ステージのローカル座標系が Z に応じて指数的にスケール・フェードされる。

## ステージと Z

表示ステージは 0 惑星 / 1 恒星系 / 2 銀河 / 3 銀河団 / 4 宇宙網。連続ズーム値 Z は整数でステージ中心に一致する(Z=0 が惑星、Z=4 が宇宙網)。各ステージの内容物は半径 O(1) のローカル単位系で構築され、focus パス上の子がローカル原点に来るよう平行移動されている(anchor)。

ステージ間の寸法比は KJ = [400, 1500, 18, 45] [調整](順に 惑星→恒星系、恒星系→銀河、銀河→銀河団、銀河団→宇宙網)。現実の比(約 10³⁻⁴, 10⁷, 10¹⁻², 10²⁻³)から圧縮されているが方向を保存しており、「世界は恒星系の中で失われ、恒星系は銀河の中で失われ、銀河同士は比較的近い」という感覚を作る。

対数スケールの区分線形補間:

    LK = KJ.map(ln), LSUM = [0, ΣLK[0..0], ΣLK[0..1], ΣLK[0..2], ΣLK[0..3]]
    L(z) = LSUM[j] + (z−j)·LK[j]   (j = clamp(floor(z), 0, 3))
    ステージ j のスケール s_j(Z) = exp(LSUM[j] − L(Z))

Z がステージを 1 進むごとに比は異なるが、操作(ピンチ・ホイール)上は等速で通過する。

Z の範囲: zMin = minStage が 0 なら −0.25(惑星へ約 4.5 倍まで寄れる)、そうでなければ minStage−0.25。zMax = 4.05 [調整だが 80-decisions.md ADR-9 の意図を保持すること]。上限が 4.05 なのは、観測者が宇宙網という媒質の外に出て「宇宙の外側」を見る神視点を禁じるためである。

Z はイージングされる: 入力(ホイール・ピンチ・タップ・ボタン)は目標値 zTarget を動かし、毎フレーム Z += (zTarget−Z)×min(1, dt×7)(prefers-reduced-motion では dt×30)。ピンチは ΔZ = ln(d0/d)/ln(2)、ホイールは deltaY×0.0016、キー ± は 0.12 刻み [調整]。

## zoomPath(focus パスの導出)[不変]

zoomPath(focusAddr) は 5 要素の anchor 配列(null 許容)を返す。CORE の純関数である。

1. 実祖先: focusAddr の segs を先頭から累積し、各累積 address を STAGE_OF_KIND = {P:0, M:0, Y:1, G:2, C:3, W:4, S:4} のステージに格納する(M は P を上書きし stage0 の anchor になる)。
2. 代表降下: focus の深さがステージ 0 に達するまで、
   deepestKind が S のとき next = neighborhood(cur).children[chainHash(cur) mod 件数](原点セル周辺の feature 列挙)、
   それ以外は childKind = DESCEND_KIND = {W:"C", C:"G", G:"Y", Y:"P"}、n = canonicalChildCount(cur, childKind)、n=0 なら打ち切り(空のボイド等)、next = cur + {k:childKind, i: chainHash(cur) mod n}。
   next のステージが未充填の場合のみ格納する(実祖先を上書きしない)。
3. スキップ階層(例 S/W/G)では該当ステージ(この例では 3)が null のまま残る。

同じ focus に対して代表降下は常に同じ対象に到達する(ズームインの再現性)。

## ステージ内容の構築

各ステージは focus 変更時に一括再構築される。子の列挙は neighborhood(anchor) の正規範囲であり、focus パス上の子が列挙に無い場合(範囲外 index)はその 1 件を追加して可視・anchor 可能に保つ。各ステージの配置ソルトと式(位置はすべて子の chain から決定):

- stage1 恒星系: 軌道半径 = 0.34 + min(segIndex,12)×0.17 + rng×0.05、角度一様、y = ±0.025×半径(salt 0x0913)。恒星は starColorOf(chain)(salt 0x57a2)の眩しい光点(芯 0.06 + ハロ 0.5 / 1.7)。惑星は極小の光点(0.013 + radius×0.005)+ 不可視ヒット領域 0.10。軌道線は存在しない。
- stage2 銀河: 全体を orientationOf(chain)(salt 0x0413)で傾ける。内部星は galaxyPointData(salt 0xB061)による N = 2600×detail 点(バルジ 24% / 薄い円盤 30% / 緩い対数螺旋腕 46%、輝度はべき乗則)。恒星系の光点は starColorOf 色のスプライト 0.042 + ヒット 0.09、位置は salt 0x6a1a(√一様 ×1.2)。detail≥8 のとき salt 0xB062 でハロ 1400 点と星団 26×34 点を追加(下位品質の点列は不変)。
- stage3 銀河団: 位置は salt 0xC1a5 の擬ガウスぶれ(×0.9, ×0.5, ×0.9)。各銀河は per-child rng(salt 0xC1a6、消費順: 不透明度→サイズ)で op = 0.35+0.65·r²、size = 0.20+0.18·r。外観は 3D で傾いた平面(galaxyPlane、傾き = orientationOf)に円形の滲みテクスチャ、opacity = op×0.45(未解像の光)。その上に、focus 銀河を除く全銀河のミニ点群(NMINI = round(170×√detail)、galaxyPointData の先頭 = 同じ星の部分集合)を per-galaxy の向き・スケール size/2.6 で単一ジオメトリに焼き込む。ヒット 0.13。
- stage4 宇宙網: セル→ステージ単位 U = 1.25。feature は webFeature の実座標に配置(色・サイズ・不透明度は 50-art.md)。ダストは 30-cosmic-web.md の近傍 + 遠方環帯 + 遠景ノード焼き込みを 1 つの点群に合流し、構造上の点の疎な部分集合(w>0.45、3 点に 1 つ、上限 700×detail)を大きく極めて淡いガス層として重ねる。

## フェードとクロスフェード [調整(構造は方式)]

    stageFade(d), d = j − Z:
      基本: d∈[−0.25, 0.85] で 1、d>0.85 で 1−(d−0.85)/0.85、d<−0.25 で 1+(d+0.25)/0.65(いずれも 0 で打ち止め)
      文脈減光: d>0.6 のとき ×(0.55 + 0.45×max(0, 1−(d−0.6)/0.4))

ステージの不透明度は素材ごとの基準値 × fade。fade ≤ 0.004 で非表示。

焦点プロキシ(各ステージで focus パス上の子を表す 1 オブジェクト)は相補クロスフェードに従う: pv = 1 − stageFade((j−1) − Z)(詳細ステージが null なら pv = 1)。詳細が減光した分だけプロキシが光を受け取り、対象は二重にも空白にもならない。さらに見かけサイズ上限: プロキシの基準スケール b に対し実スケール = min(b, 0.26/s)。ステージが拡大されている間は遠くの小さな光に留まり、s→1 で本来のマーカーサイズへ滑らかに戻る。

## 点群の規律 [方式]

sizeAttenuation 有効の点群は 2 種に分類される(素材の userData.scenery フラグ)。

- entered(原点中心の媒質: 銀河内部、宇宙網ダスト): 点サイズ = 基準 × min(s, 1)(拡大方向に成長しない)。s > 1.5 で aggF = 1 − ln(s/1.5)/ln(20) により溶解し、細部を下のステージへ引き継ぐ。統計的な滲みは物体のように拡大されない、という原則の実装である。
- scenery(遠くの実体: 銀河団のミニ点群): 点サイズ = 基準 × s。位置とサイズが同率で伸縮するため角サイズ不変で、通常の物体として振る舞い、溶解しない。

## 中から見た空 [方式/調整]

惑星・恒星系ビューの背景は装飾ではなく、所属銀河の中から見た空として focus ごとに合成される(buildSky)。物理的な内部投影ではなく、決定論的な印象合成である(ADR-12)。

- 乱数は salt 0x5C1E(系の chain 基点)。半径 R = 150 の天球。
- 天の川の帯の法線 = orientationOf(所属銀河の chain)。つまり帯の向きは、ズームアウト後に外から見る銀河円盤の傾きと厳密に一致する。
- 星は二層: 微光 field(N = min(60000, 12000×detail^0.78)、帯バイアス 0.62、輝度 0.10+0.90·r^2.4、サイズ 1.15×pixelRatio)と、解像された明るい星(N = 280×√detail、バイアス 0.5、輝度 0.4+0.6·r^1.4、サイズ 2.3×pixelRatio)。sizeAttenuation:false の点サイズはデバイスピクセル単位のため、pixelRatio を掛けなければ Retina で半減する(既知の落とし穴)。
- 帯の霞: 帯に沿った 14 枚の淡い発光スプライト(銀河 tint、opacity 0.055+0.03·r)。
- 隣接銀河: 銀河団の実データから最大 4 つ、salt 0x5C1F で方向・回転・縦横比を決めた極めて淡い滲み(opacity 0.10+0.10·r)。
- skyFade(z) = clamp(1 − (z−1.2)/0.6): 銀河スケールへ引くにつれ内側の空は溶けて消え、外から見た銀河へ引き継がれる。

## カメラと操作 [調整]

PerspectiveCamera(fov 50, near 0.05, far 200000)。カメラは原点を注視し距離 3.6 固定、球面座標 (theta, phi) のみ操作(phi は [0.15, π−0.15] にクランプ)。1 本指ドラッグ = 回転(感度 0.005/px)、離すと慣性(速度は 0.65/0.35 の指数平滑で追跡、減衰 exp(−3.2·dt)、閾値 0.002 で停止)。2 本指ピンチ = Z。タップ判定は移動 8px 未満かつ 400ms 未満で、可視ステージ(fade > 0.3)の pickable にレイキャストする。far=200000 は K_j 下の可視最大距離(約 5×10⁴)を覆うために必要で、遠方は加算・depthWrite なし素材のみのため深度精度の問題は生じない。
