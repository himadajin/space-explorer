# 80 — decisions: 設計判断の根拠(ADR)

現在の設計がなぜこの形なのかを、判断ごとに記録する。形式は 決定 / 理由 / 棄却した代替 / 帰結。歴史ではなく、いま有効な根拠として現在形で書く。仕様の本文と矛盾した場合は仕様本文が優先し、本書を追記修正する。

## ADR-1 単一 HTML の artifact

決定: 単一 HTML、Three.js を CDN から読み込み、ビルドなし、永続化なし。
理由: そのままリンク共有でき、閲覧者の環境構築が不要。
棄却: リポジトリ + ビルド構成(共有の摩擦が目的に合わない)。
帰結: 状態の持ち越しは address 文字列のみ。モジュール境界はコメント区分で規律する。移植時の指針は 90-roadmap.md。

## ADR-2 identity はハッシュ連鎖、乱数は mulberry32

決定: address の各 segment を splitmix 系 combine で順に混ぜた 32bit 状態を種とし、mulberry32 で乱数列を引く。fingerprint は量子化パラメータの連鎖ハッシュ hex8。
理由: ローカル評価のみで任意地点を再生成でき、検証が数値比較で済む。
棄却: 64bit ハッシュや暗号ハッシュ(JS の 32bit 整数演算で十分、速度優先)。
帰結: 生成関数の乱数消費順が identity の一部になる(20-generation.md 冒頭の警告)。

## ADR-3 正規個数と範囲外 index の共存

決定: 候補列挙は親ハッシュ由来の正規個数に限定し、jump は任意 index を解決可能のまま残す。
理由: 「未訪問にも address で行ける」到達性の不変条件と、「見えるものは実在の構成」の両立。
棄却: 範囲外 index の拒否(到達性を壊す)/ 無制限の列挙(構成が無意味になる)。
帰結: 範囲外対象は幾何配置を欠くことがある(webFeature.exists = false)が identity は等価。

## ADR-4 手帳は座標であってセーブデータではない

決定: ブックマークは {address, fingerprint} のみ。閉じれば消える。まとめて copy / 貼り戻しで持ち帰る。自動履歴・自由な命名は持たない。
理由: 宇宙は決定論的で保存不要。失われうるのはメモだけ、という世界観が artifact の揮発性を欠陥ではなく仕様にする。書き留める行為の重みは自動記録が薄める。
棄却: localStorage(artifact で不可)、履歴の自動記録、命名機能(初期版では雰囲気を壊すスマホ入力を伴う)。
帰結: 永続化の責任は詩的にユーザーの手帳(メモアプリ)へ委ねられる。

## ADR-5 ズームはステージ積層、カメラは動かない

決定: 5 表示ステージを各々ローカル単位系で構築し、K^(j−Z) 型の指数スケーリングとフェードで合成する。カメラは原点注視・距離固定。
理由: 浮動小数点精度(巨大座標の破綻)を避け、任意スケール間の連続遷移を単純な毎フレーム変換で得る。
棄却: 単一座標系での実カメラ移動(精度と LOD 管理が破綻)。
帰結: focus 変更 = ステージ一括再構築。anchor(focus パス上の子を原点に置く)が位置連続性を担う。

## ADR-6 ステージ間比率はステージごとに異なる(K_j)

決定: KJ = [400, 1500, 18, 45]。現実比から圧縮しつつ方向を保存する。
理由: 一様比(旧 25)は全スケールをミニチュア感にする。現実は惑星系側が桁違いに疎で、銀河同士は比較的近い。完全な現実比はズームが虚無の通過になる。
棄却: 一様 K / 完全現実比。
帰結: Z は区分線形の対数補間で等速操作を保つ。遠クリップ面は 2×10⁵ が必要。

## ADR-7 点群の規律(entered / scenery)

決定: 原点中心の媒質(銀河内部・ダスト)はサイズ上限 + 進入時に溶解。遠くの実体(銀河団のミニ点群)はフレームと同率でスケールし溶解しない。
理由: 集合点は統計的な滲みであり物体のように拡大されてはならない。一方、off-origin の実体まで溶かすと「見せたい瞬間に消える」誤爆を起こす。
棄却: 全点群への一律規律(どちらか一方に倒すと巨大な四角い点、または兄弟の消失を生む)。
帰結: 素材フラグ userData.scenery による二分。焦点銀河はミニ点群から除外(実点群が同じ星を担う)。

## ADR-8 フィラメントは線として描かない

決定: 宇宙網の構造は密度サンプリングされたダストのみで見せる。線分・軌道線などの図解要素は描かない。
理由: 実際の宇宙に線は存在せず、線は SF マップの記号である。構造は点の密度が語る。
棄却: 淡い線分(囁き程度に残す案も、図解感が消えないため全廃)。
帰結: ダストが構造表現の主役となり、点数と密度コントラストが画質を決める。フィラメントの列挙自体は destination として維持される。

## ADR-9 観測者は媒質の内側にいる

決定: ダストは ±2 セル + 球状フェザリング、遠方環帯と遠景焼き込みで奥へ続く気配を置き、ズーム上限 4.05 で媒質の外に出る神視点を禁じる。
理由: 有限の描画データが「宇宙の外側」として知覚される誤解を、無限実装なしで断つ。闇は世界の果てではなく視程の限界である。
棄却: 真の無限化(カメラ並進 + セルの逐次生成。focus 原点固定の全設計と衝突し、現時点で移動手段も存在しない。90-roadmap.md)。
帰結: 宇宙網全体の俯瞰ショットは仕様として撮れない。

## ADR-10 光の設計(二層・無彩色・べき乗則・ACES)

決定: 50-art.md の原理群。
理由: 参照は深宇宙写真であり、ゲーム画面ではない。単層グラデーションはピンボケに、記号色はネオンに、一様輝度は人工物に見える。
棄却: 発光の強い SF 調。
帰結: 種類の区別は色相でなく明るさと形で行う。表示の脱色は識別パラメータに触れない。

## ADR-11 共有導出(親子整合)

決定: 二つのスケールに現れる性質は単一の導出から引く。銀河の向き、恒星色、衛星の継承、銀河星の部分集合。
理由: 「遠くで見た光に近づくと、その通りのものがある」という体験の信頼が探索の核だから。
棄却: スケールごとの独立生成(近づくと別物になる)。
帰結: 新表現の追加時は既存 chain / 安定パラメータからの導出を最初に検討する規範。

## ADR-12 中から見た空は合成であって投影ではない

決定: 惑星からの夜空は、所属銀河のパラメータ(向き・色)と実在の隣接銀河から決定論的に合成する。内部からの厳密な投影はしない。
理由: 厳密投影のコストは体験の増分に見合わない。向きと色の一致だけで「同じ宇宙にいる」連続性は成立する。
棄却: 銀河点群の全天レイトレース的投影。
帰結: 同じ惑星では常に同じ空。銀河スケールで内側の空は溶暗し、外観に引き継ぐ。

## ADR-13 品質 4 段、still は確認の先、安全網つき

決定: std / high / xhigh をボタン巡回、still(×8 + pixelRatio 3)は専用ボタン + 確認パネル経由。切替後の性能監視で自動降格、コンテキスト喪失で std 復帰。
理由: フレームレートより一枚絵を優先する選択肢を、うっかりでは選べないが見える場所に置く(見えない機能は使われない)。失敗の最悪着地を std に固定して攻めた絵作りを可能にする。
棄却: 長押しなどの隠し導線 / 品質の自動のみ。
帰結: 品質は表示専用(不変条件 9)。同一品質での決定性は保つが、品質間で点の配置は異なってよい。

## ADR-14 起動は宇宙網スケール

決定: 起動時の focus は初期惑星のまま、初期 Z = 4。
理由: 最初に見えるのが宇宙の大規模構造であるほうが広さを体験でき、ズームインが「全景から一つの世界へ降りる」導入になる。
棄却: 惑星から開始(初期の実装)。
帰結: identity は不変。初回の宇宙網構築コストが起動時に前倒しされる。

## ADR-15 旧列挙 address の互換方針

決定: 実在列挙に存在しない feature を指す address も identity として解決し続ける(配置のみ欠く)。
理由: 不変条件 1(同じ address → 同じ対象)は列挙方式の変更より優先される。既存の手帳を殺さない。
帰結: webFeature.exists による配置有無の分岐が UI 側に存在する。
