# 90 — roadmap: 先送りした構想と移植の方針

本書は仕様ではなく、検討済みだが未実装の構想と、環境移行時の対応方針を記す。実装状況と混同しないこと。

## 先送りした構想

宇宙網の真の無限化。現在は「観測者は媒質の内側」(ADR-9)の演出で有限データを覆っている。真の無限化には、宇宙網スケールでの連続的な並進移動と、移動に応じたセルの逐次生成・破棄(re-centering)が必要で、focus 原点固定の現設計(ADR-5)全体に波及する。宇宙網を「漂う」操作を導入する時に併せて設計し直すのが妥当である。生成系(種点・feature・ダスト)はローカル評価のみなので、この拡張を妨げない。

天体への命名。手帳の各項目に自由なラベルを付ける機能。ADR-4 の通り初期版では意図的に外している。導入するなら、名前は identity に含めない(表示メモに留める)こと。

address の短縮共有形式。現在の文字列は正確だが長い。共有向けに base 変換等の短縮形を作る場合、正規文字列との相互変換のみで実装し、chainHash の入力(正規形)を変えないこと。

宇宙網スケールでの移動手段。現在の移動は feature タップと jump のみ。ドラッグでの平行移動を入れる場合、上記の無限化とセットになる。

seed の切り替え UI。identity 上は S segment で既に表現できる(jump で別 seed に跳べる)。UI として seed を編集する導線は未実装。

## Vite + React + TypeScript への移植方針

現アーティファクトは移植を想定した構造になっている。対応は次の通り。

- CORE(CORE-BEGIN / CORE-END 間)は依存ゼロの純関数群であり、そのまま TypeScript モジュール(例 src/core/)に移せる。10〜30 の仕様が型定義の根拠になる。fingerprint の既知の正解値(70-verification.md)が移植の一致判定になる。JavaScript の 32bit 整数演算(`>>> 0`, `Math.imul`)と `Math.round` の半端値挙動をそのまま保つこと。
- 回帰テストは Node ハーネスから vitest 等へ機械的に移せる。テスト項目一覧は 70-verification.md。
- render 層(Three.js)は 1 コンポーネント(canvas 保持)+ 命令的モジュールとして移すのが素直で、React の宣言的レンダリングに乗せ替える必要はない。ステージ再構築・毎フレーム変換(40-zoom.md)は命令的なまま保つ。
- UI 層(HUD・手帳・パネル)は React 化の恩恵が大きい部分。state(60-ui-state.md)を store に写像する。
- localStorage が使える環境になっても、手帳の揮発性を仕様として維持するか(ADR-4 の世界観)、永続化をオプトインにするかは、その時点での意思決定として ADR に追記する。
- Three.js のバージョンを r128 から上げる場合、点サイズ・トーンマッピング・色空間(r152 以降の colorSpace 既定変更)の挙動差で絵が変わる。50-art.md の基準値を再チューニングの出発点とし、identity には影響しないことをテストで確認する。

## 既知の制約(現状のまま受け入れているもの)

- 撮影の保存はサンドボックスのダウンロード許可に依存する。
- still への切替は約 2 秒の同期的再構築を伴う(veil と rendering 表示で覆う)。
- 銀河平面は厚みゼロのため、真横からは一瞬視認できない(ヒット領域は維持される)。
- 品質段階が異なるとダスト等の点配置は異なる(同一品質内では決定論的)。
